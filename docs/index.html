Understand Kafka Ordering
in 3 Levels

A progressive, hands-on demo that teaches you how Kafka handles message ordering — and how to get both speed and order in production systems.

View on GitHub

The Problem

You're building an e-commerce system. Each order goes through a lifecycle:

CREATED → PAID → SHIPPED → DELIVERED

Processing these events out of order is catastrophic — you can't ship before payment, and you can't deliver before shipping. How does Kafka help you maintain this order at scale?

The Three Levels

Level 1

Single Partition — Guaranteed Order

With one partition, Kafka is an append-only log. Messages come out in the exact order they go in. Simple and correct, but limited throughput.

Ordered 1 partition 1 thread
Level 2

Multi-Partition + Threads — Speed, No Order

We add 4 partitions and 4 producer threads. Throughput goes way up, but without a partition key, Kafka round-robins messages — and order breaks.

Unordered 4 partitions 4 threads
Level 3

Consistent Hashing — Speed AND Order

We use order_id as the partition key. Kafka hashes the key to deterministically route all messages for the same order to the same partition.

Ordered per key 4 partitions 4 threads

How Consistent Hashing Works

┌──────────────┐ ┌───►│ Partition 0 │───► Consumer 0 (ORD-002, ORD-006) │ ├──────────────┤ Producer ──┼───►│ Partition 1 │───► Consumer 1 (ORD-003, ORD-008) (key= │ ├──────────────┤ order_id) ├───►│ Partition 2 │───► Consumer 2 (ORD-001, ORD-005) │ ├──────────────┤ └───►│ Partition 3 │───► Consumer 3 (ORD-004, ORD-009) └──────────────┘ hash("ORD-001") % 4 = 2 ← always partition 2 hash("ORD-002") % 4 = 0 ← always partition 0

Quick Start

  1. Start Kafka 
    docker compose up -d
  2. Create Topics 
    ./setup_topics.sh
  3. Build the Project 
    mvn clean compile
  4. Run a Demo (e.g., Level 3) 
    # Terminal 1
mvn exec:java -Dexec.mainClass="com.demo.kafka.level3.Producer"

# Terminal 2
mvn exec:java -Dexec.mainClass="com.demo.kafka.level3.Consumer"

Key Takeaways

Level Partitions Ordering Throughput Key Concept
1 1 Total Low Single partition = total order
2 4 Broken High No key = round-robin = chaos
3 4 Per-key High Partition key = speed + order
